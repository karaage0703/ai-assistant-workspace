# スキル作成

自分専用のスキルを作成・改善する。

## スキルとは

スキルは「AIへの指示書」。SKILL.mdファイルに手順を書いておくと、AIがその通りに動いてくれる。

**メリット:**
- 毎回同じ説明をしなくていい
- 複雑な手順を自動化できる
- チームで共有できる

## スキルの4分類

| 分類 | 特徴 | 例 |
|------|------|-----|
| 対話型 | 会話しながら成果物を作る | ブログ記事生成 |
| 自動実行型 | 指示一つで完結 | ニュース取得 |
| ツール連携型 | 外部ツールと連携 | YouTube字幕取得 |
| 思考支援型 | 考えを整理する | ブレスト支援 |

## スキル作成手順

### Step 1: ヒアリング

以下を確認：
1. **何をするスキル？**
2. **どんな時に使う？**
3. **入力と出力は？**
4. **スクリプトは必要？** — Python/Bash等の自動化有無

### Step 2: 分類と設計

4分類のどれに該当するか判定し、適切な構成を選択。

### Step 3: ディレクトリ作成

```bash
mkdir -p skills/<skill-name>
```

### Step 4: SKILL.md作成

以下のテンプレートを使う：

```markdown
---
name: <skill-name>
description: [50-200文字。何をするか＋いつ使うか＋トリガーフレーズ]
---

# [スキル名]

[1行の説明]

## 手順

### Step 1: [最初のステップ]

[具体的な作業内容]

### Step 2: [次のステップ]

[具体的な作業内容]

### Step 3: [出力]

[出力フォーマット]

## 使用例

[呼び出し例]
```

**パス規則:**
- ハードコードパス禁止（ワークスペースの場所に依存するパスを書かない）
- 相対パスを使う（`skills/<name>/` や `../../`）

### Step 5: テスト

作ったスキルを実際に呼び出して動作確認。

### Step 6: AGENTS.mdのスキル一覧に追加

`AGENTS.md` の「スキル一覧」セクションに新しいスキルを追記。

### Step 7: 完了報告

```
スキル作成完了！

skills/<skill-name>/
- SKILL.md
```

## 良いスキルを作るコツ

### descriptionは具体的に

**良い例:**
```
YouTubeの動画URLから字幕を取得してノートにまとめる。「YouTube動画をまとめて」で使用。
```

**悪い例:**
```
動画をまとめる
```

### 手順は明確に

- Step形式で番号付き
- コマンドがあれば具体的に書く
- 曖昧な表現を避ける

### 出力フォーマットを決める

期待する出力の形を示しておくと、毎回同じ品質になる。

### チェックリスト

- [ ] name と description がフロントマターにある
- [ ] description は50-200文字で、トリガーフレーズを含む
- [ ] 手順がStep形式で明確
- [ ] コマンドがある場合は具体例つき
- [ ] AGENTS.md のスキル一覧に追加した
